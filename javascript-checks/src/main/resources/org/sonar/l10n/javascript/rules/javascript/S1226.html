虽然从技术上讲，从函数体内部分配参数是正确的，但这降低了代码的可读性，因为开发人员无法在不遍历整个函数的情况下判断原始参数或某个临时变量是否被访问。此外,一些 开发人员还可能希望函数参数的分配对调用者可见，但实际情况并非如此，这种不可见性可能会使调用者感到困惑。相反，所有参数、捕获的异常和foreach参数都应该作为常量处理。

不合规的代码示例

function MyClass(name, strings) {
  name = foo;                    // 不合规

  for (var str of strings) {
    str = "";  // 不合规
  }
}
