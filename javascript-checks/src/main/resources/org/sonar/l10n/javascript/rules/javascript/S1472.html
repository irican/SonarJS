由于语句末尾的分号是可选的，因此在单独的行上开始函数调用参数会使代码混乱。 它可能会导致错误，并且 will 很可能导致维护者的问题。

开发人员最初的意图是什么?

  1. 定义一个函数，然后在闭包中执行一些不相关的代码?
  2. 将第二个函数作为参数传递给第一个函数?

第一个选项将由JavaScript解释器选择。

通过扩展，为了提高可读性，任何类型的函数调用参数都不应该从新行开始。

不合规的代码示例

var fn = function () {
  //...
}

(function () { // 不合规
  //...
})();

合规的解决方案

Either

// 定义一个函数
var fn = function () {
  //...
}; // <-- semicolon added

// 然后在闭包中执行一些代码
(function () {
  //...
})();

Or 

var fn = function () {
  //...
}(function () { // <-- 在同一行启动函数调用参数
  //...
})();
