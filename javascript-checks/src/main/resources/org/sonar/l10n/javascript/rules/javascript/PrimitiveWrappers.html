将包装器对象用于基本类型是不必要的、令人困惑的和危险的。如果使用包装器对象构造函数进行类型转换，只需删除 new 关键字，就会自动得到一个原始值。 如果使用包装器对象作为向原语添加属性的方法，则应该重新考虑设计。这样的使用被认为是不好的实践，应该重构。

不合规的代码示例

let x = new Number("0");
if (x) {
  alert('hi');  // 显示 'hi'.
}

合规的解决方案

let x = Number("0");
if (x) {
  alert('hi');
}

例外

当基本类型构造函数的参数是同一类型的文字时，将忽略这些情况，除了 new Boolean(false).

let booleanObject = new Boolean(true);
let numberObject = new Number(0);
let stringObject = new String('');
