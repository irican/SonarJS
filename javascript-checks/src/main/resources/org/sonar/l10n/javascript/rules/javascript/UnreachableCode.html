跳转语句(return, break 和 continue) 和 throw将控制流移出当前代码块。因此，跳转之后的任何语句都是死代码。

不合规的代码示例

function fun(a) {
  var i = 10;
  return i + a;
  i++;             // 不合规; 这是绝对不会执行的
}

合规的解决方案

function fun(int a) {
  var i = 10;
  return i + a;
}

例外

该规则忽略了switch子句中的不可达break语句。

switch (x) {
  case 42:
     return 43;
     break;   // 合规的
  default:
    doSomething();
}

没有初始化的挂起变量声明总是被认为是可访问的。

function bar() {
  return x = function() {
    x.foo = 42;
  }
  var x;
}
