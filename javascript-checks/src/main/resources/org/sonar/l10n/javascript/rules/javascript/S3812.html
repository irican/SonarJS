混淆操作顺序几乎总是会产生意想不到的结果。

同样，错误的否定也会产生不好的结果。例如，考虑一下 !key in dict 和 !(key in dict)之间的区别。第一个查找 dict 中的布尔值 (!key) ,另一个查找字符串并反转结果。 !obj instanceof SomeClass 也有同样的问题。

当in 或instanceof 的左操作数被否定时，该规则会引发一个问题。

不合规的代码示例

if (!"prop" in myObj) {  // 不合规;  "in" 操作符正在检查属性 "false"
  doTheThing();  // 这个块将永远不会执行
}

if (!foo instanceof MyClass) {  // 不合规; "!foo" 返回一个布尔值，它不是任何东西的实例
  doTheOtherThing();  // 这个块永远不会执行

合规的解决方案

if (!("prop" in myObj)) {
  doTheThing();
}

if (!(foo instanceof MyClass)) {
  doTheOtherThing();
}
