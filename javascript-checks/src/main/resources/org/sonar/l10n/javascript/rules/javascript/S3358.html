仅仅因为你能做某件事，并不意味着你应该做，这就是嵌套三元运算的情况。 嵌套三元运算符导致的代码在编写时看起来很清晰，但是六个月后，维护者(或者更糟的是，将来的您)会感到困惑和想要痛骂。

相反，为了清晰起见，应该使用另一行来将嵌套操作表示为单独的语句.

不合规的代码示例

function getTitle(p) {
  return p.gender == "male" ? "Mr. " : p.isMarried() ? "Mrs. " : "Miss ";  // 不合规
}

合规的解决方案

function getTitle(p) {
  if (p.gender == "male") {
    return "Mr. ";
  }
  return p.isMarried() ? "Mrs. " : "Miss ";
}
