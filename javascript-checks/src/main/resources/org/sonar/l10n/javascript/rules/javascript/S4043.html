许多JavaScript的Array方法在保持源数组不变的情况下返回数组的修改版本。reverse 和 sort不属于这个类别。相反，它们 处除了 返回修改后的版本之外，还修改源数组， 这可能不是我们想要的。

当分配这些方法的返回值时，该规则会引发一个问题，这可能导致维护人员忽略原始值被更改的事实。

不合规的代码示例

var b = a.reverse(); // 不合规
var d = c.sort(); // 不合规

合规的解决方案

var b = [...a].reverse();  // 反结构，并创建一个新的数组，所以反向不会影响'a'
a.reverse();

c.sort(); // 这将对数组进行排序
