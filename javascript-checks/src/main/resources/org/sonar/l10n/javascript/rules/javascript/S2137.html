JavaScript有特殊的标识符，虽然没有保留，但仍然不能用作标识符。它们包括:

  • eval - 将字符串计算为JavaScript代码
  • arguments - 用于通过索引属性访问函数参数。
  • undefined - 返回尚未分配的值和属性
  • NaN - 不是一个数字;当数学函数失败时返回。
  • Infinity - 当一个数字超过浮点数的上限时

不应该绑定或分配这些单词，因为这样做会覆盖这些标识符的原始定义。此外，在JavaScript严格模式代码中，分配或绑定其中一些名称将生成错误。

不合规的代码示例

eval = 17; // 不合规
arguments++; // 不合规
++eval; // 不合规
var obj = { set p(arguments) { } }; // 不合规
var eval; // 不合规
try { } catch (arguments) { } // 不合规
function x(eval) { } // 不合规
function arguments() { } // 不合规
var y = function eval() { }; // 不合规
var f = new Function("arguments", "return 17;"); // 不合规

function fun() {
  if (arguments.length == 0) { // 合规
    // do something
  }
}

合规的解决方案

result = 17;
args++;
++result;
var obj = { set p(arg) { } };
var result;
try { } catch (args) { }
function x(arg) { }
function args() { }
var y = function fun() { };
var f = new Function("args", "return 17;");

function fun() {
  if (arguments.length == 0) {
    // do something
  }
}
