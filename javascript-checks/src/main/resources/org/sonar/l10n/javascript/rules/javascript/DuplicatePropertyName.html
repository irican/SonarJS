JavaScript允许在类和对象文本中使用重复的属性名，但只有重复名称的最后一个实例才确定将为其使用的实际值。因此，更改重复名称的其他出现的值将没有效果，并可能导致误解和错误。

使用重复的 constructor定义一个class 将产生一个错误。

在ECMAScript 2015之前，使用重复的名称会在JavaScript严格模式代码中产生错误。

不合规的代码示例

var data = {
  "key": "value",
  "1": "value",
  "key": "value", // 不合规 - 重复的 "key"
  'key': "value", // 不合规 - 重复的 "key"
  key: "value", // 不合规 - 重复的 "key"
  \u006bey: "value", // 不合规 - 重复的 "key"
  "\u006bey": "value", //不合规 - 重复的 "key"
  "\x6bey": "value", // 不合规 - 重复的 "key"
  1: "value" // 不合规 - 重复的 "1"
}

合规的解决方案

var data = {
  "key": "value",
  "1": "value",
  "key2": "value",
  'key3': "value",
  key4: "value",
  \u006bey5: "value",
  "\u006bey6": "value",
  "\x6bey7": "value",
  1b: "value"
}
