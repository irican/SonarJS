arguments.caller 和 arguments.callee 使许多的优化是不可能的，所以它们在最新版本的JavaScript中被禁用。 事实上，根据文档，EcmaScript 5禁止在strict模式下使用这两种方法:

严格模式函数的参数对象定义了名为“caller”和“callee”的不可配置访问器属性，这些属性在访问时抛出一个类型错误异常。

同样的限制也适用于函数在strict模式的 caller 和 arguments 属性。

不合规的代码示例

function whoCalled() {
   if (arguments.caller == null)   //不合规
      console.log('I was called from the global scope.');
   else
      console.log(arguments.caller + ' called me!');  // 不合规

  console.log(whoCalled.caller);  // 不合规
  console.log(whoCalled.arguments);  // 不合规
}
