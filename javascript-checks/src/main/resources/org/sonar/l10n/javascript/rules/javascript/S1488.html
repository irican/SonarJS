声明一个变量只是为了立即返回或抛出它是一个不好的做法。

一些开发人员认为，该实践提高了代码的可读性，因为它使他们能够显式地命名返回的内容。 但是，该变量是一个内部实现细节，不公开给方法的调用者。方法名应该足以让调用者确切地知道将返回什么。

注意，此规则要求Node.js在分析期间可用。

不合规的代码示例

function computeDurationInMilliseconds() {
  var duration = (((hours * 60) + minutes) * 60 + seconds ) * 1000 ;
  return duration;
}

合规的解决方案

function computeDurationInMilliseconds() {
  return (((hours * 60) + minutes) * 60 + seconds ) * 1000 ;
}
