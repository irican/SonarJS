JavaScript的神奇之处在于，您可以将参数传递给不声明参数的函数，另一方面，您可以在no-args 函数中使用这些传入的参数。

但仅仅因为你能，并不意味着你应该。函数中未显式声明参数的期望和使用会使调用者感到困惑。没有人必须阅读并完全理解一个函数才能胜任地使用它。

如果不想显式地命名参数，请使用…语法，指定需要一个可变数量的参数。然后在函数内部，您将处理一个一流的数组，而不是类似数组的结构。

不合规的代码示例

function concatenate() {
  let args = Array.prototype.slice.call(arguments);  // 不合规
  return args.join(', ');
}

function doSomething(isTrue) {
  var args = Array.prototype.slice.call(arguments, 1); // 不合规
  if (!isTrue) {
    for (var arg of args) {
      ...
    }
  }
}

合规的解决方案

function concatenate(...args) {
  return args.join(', ');
}

function doSomething(isTrue, ...values) {
  if (!isTrue) {
    for (var value of values) {
      ...
    }
  }
}
