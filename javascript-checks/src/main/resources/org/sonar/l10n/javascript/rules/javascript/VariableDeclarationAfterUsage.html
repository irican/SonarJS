用var声明的变量具有特殊的属性，无论它们在函数中声明在哪里，它们都会“浮动”到函数的顶部，甚至在声明之前就可以使用。这使得范围划分很混乱，尤其是对于新程序员来说。 为了将混淆降到最低，var声明应该发生在它们声明的变量第一次被使用之前。

不合规的代码示例

var x = 1;

function fun(){
  alert(x); // 不合规，因为x稍后将在同一范围内声明
  if(something) {
    var x = 42; // 函数作用域中的声明(不是块作用域!）
  }
}

fun(); // 意外地警告“未定义”而不是“1”

合规的解决方案

var x = 1;

function fun() {
  print(x);
  if (something) {
    x = 42;
  }
}

fun(); // 打印 "1"
