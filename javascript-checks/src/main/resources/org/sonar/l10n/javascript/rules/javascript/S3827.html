当引用一个不存在的变量时，会引发一个ReferenceError。

由于JavaScript的动态特性，这可能发生在许多场景中:

  • 在符号名称中键入错误时。
  • 当使用声明前使用let或const声明的变量时(不像var-declaration，它们不会被提升到作用域的顶部)。
  • 由于混淆了let-和const-声明的作用域(它们具有块作用域，不同于var-声明，它们具有函数作用域)。
  • 当访问错误范围内的属性时(例如忘记指定this.)。

此规则不会对使用 sonar.javascript.globals和 sonar.javascript.environments 属性定义的全局变量产生问题。

不合规的代码示例

var john = {
  firstName: "john",
  show: function() { console.log(firstName); } // 不合规: 没有定义firstName
}
john.show();

合规的解决方案

var john = {
  firstName: "john",
  show: function() { console.log(this.firstName); }
}
john.show();
