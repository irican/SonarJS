当一个变量被分配一个 undefined or null 值时，它没有属性。无论如何，试图访问这样一个变量的属性都会导致 TypeError, 如果在catch块中没有捕捉到错误，则会导致脚本的突然终止。但是，与其捕获这种情况下的 catch-ing ，最好完全避免它。

不合规的代码示例

if (x === undefined) {
  console.log(x.length); // 不合规; 将抛出TypeError
}

请参阅

  • MITRE, CWE-476 - NULL Pointer Dereference
  • CERT, EXP34-C. - Do not dereference null pointers
  • CERT, EXP01-J. - Do not use a null in a case where an object is required
