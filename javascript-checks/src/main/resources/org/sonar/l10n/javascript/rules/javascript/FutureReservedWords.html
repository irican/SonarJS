在未来的语言发展过程中，可以使用以下单词作为关键字，因此应避免使用它们作为标识符，以便更容易地采用这些潜在的未来版本:

  • await
  • class
  • const
  • enum
  • export
  • extends
  • implements
  • import
  • interface
  • let
  • package
  • private
  • protected
  • public
  • static
  • super
  • yield

使用这些单词作为标识符会在JavaScript strict 模式代码中产生错误。

不合规的代码示例

var package = document.getElementsByName("foo"); // 不合规
var someData = { package: true };                 // 合规,因为这里不使用它作为标识符

合规的解决方案


var elements = document.getElementsByName("foo"); // 合规
