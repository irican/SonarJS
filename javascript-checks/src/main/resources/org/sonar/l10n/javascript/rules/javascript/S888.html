使用相等运算符(== and !=)测试for 循环终止是危险的， 因为它可以建立一个无限循环。相反，使用更广泛的关系运算符可以投射更广泛的网络，从而使意外地编写无限循环变得更困难(但并非不可能)。

不合格的代码示例

for (var i = 1; i != 10; i += 2)  // 不合规. 无限的;i从9直接到11。
{
  //...
}

合规的解决方案

for (var i = 1; i <= 10; i += 2)  // 合规
{
  //...
}

例外

如果循环计数器在循环体中没有修改，则忽略相等运算符，并且:

  • 从结束值以下开始，每次迭代递增1。
  • 从结束值以上开始，每次迭代递减1。

当测试针对 null时，相等运算符也会被忽略。

for (var i = 0; arr[i] != null; i++) {
  // ...
}

for (var i = 0; (item = arr[i]) != null; i++) {
  // ...
}
