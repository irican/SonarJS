ECMAScript 2015引入了用于块范围变量声明的let和const关键字。使用const创建只读(常量)变量。

由var创建的变量类型与由let创建的变量类型之间的区别是很重要的，切换到let将有助于缓解许多变量范围问题，这些问题在过去曾引起混淆。

因为这些新的关键字创建了更精确的变量类型，所以它们在支持ECMAScript 2015的环境中更受欢迎。但是，从var切换到let可能需要进行一些重构，您应该知道，在ecmascript 2015之前的环境中，它们会引发SyntaxErrors。

当使用var而不是const或let时，该规则会引发一个问题。

不合规的代码示例

var color = "blue";
var size = 4;

合规的解决方案

const color = "blue";
let size = 4;
