if语句的条件始终为false，则会使代码块失去功能。if 语句的条件始终为true，则完全是冗余的，这会降低代码的可读性。

出现这些守则的原因有三:

  • 调试期间更改了if语句，并且提交了调试代码。
  • 有些值没有设置。
  • 有些逻辑并没有做程序员认为它做的事情。

在这些情况中， 无条件if 语句应该被删除。

不合规的代码示例

if (true) {  // 不合规
  doSomething();
}
...
if (false) {  // 不合规
  doSomethingElse();
}

if (!options || options === true) { doThirdThing(); }  // 不合规; 总是为 true

合规的解决方案

doSomething();

doThirdThing();

请参阅

  • MITRE, CWE-489 - Leftover Debug Code
  • MITRE, CWE-570 - Expression is Always False
  • MITRE, CWE-571 - Expression is Always True
  • MISRA C:2004, 13.7 - Boolean operations whose results are invariant shall not be permitted.
  • MISRA C:2012, 14.3 - Controlling expressions shall not be invariant

弃用

这条规则是不宜用的;使用 {rule:javascript:S2583} 。
