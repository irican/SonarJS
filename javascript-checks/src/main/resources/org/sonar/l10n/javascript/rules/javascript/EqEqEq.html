== 和 != 操作符在比较值之前执行类型强制。这很糟糕，因为它可以掩盖类型错误。 例如，它计算 ' \t\r\n' == 0 为 true.

最好总是使用无副作用的 === 和 !== 操作符。

不合规的代码示例

if (var == 'howdy') {...} // 不合规

合规的解决方案

if (var === 'howdy') {...}

例外

即使测试变量对null的相等性并不能像大多数JavaScript开发人员认为的那样，使用== 或 != 在这种上下文中是可容忍的。在下面的例子中，如果foo 没有被初始化，那么它的默认值不是 null 而是 undefined.尽管如此， undefined == null, 因此JavaScript开发人员获得了预期的行为。

if(foo == null) {...}
