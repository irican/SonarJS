当 if 语句后面跟着一个或多个 else if 语句时，应用此规则;最后一个 else if 应该后跟一个 else 语句。

对最终 else 语句的要求是防御性编程。

else 语句应该采取适当的操作，或者包含关于为什么不采取操作的适当注释。这与在switch语句中使用最终的default 子句的要求是一致的。

不合规的代码示例

if (x == 0) {
  doSomething();
} else if (x == 1) {
  doSomethingElse();
}

合规的解决方案

if (x == 0) {
  doSomething();
} else if (x == 1) {
  doSomethingElse();
} else {
  throw "Unexpected value for x";
}

  • MISRA C:2012, 15.7 - All if...else if constructs shall be terminated with an else statement
  • CERT, MSC01-C. - Strive for logical completeness
  • CERT, MSC57-J. - Strive for logical completeness
