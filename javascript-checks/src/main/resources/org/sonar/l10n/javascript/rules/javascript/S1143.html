使用来自 finally 块的return, break, throw, 和 continue覆写来自挂起的try 和 catch 块的类似语句。

当一个跳转语句 (break, continue, return 和 throw) 强制控制流离开一个 finally 块时，这个规则就会产生一个问题。

不合规的代码示例

function foo() {
    try {
        return 1; // 期望返回1
    } catch(err) {
        return 2; // 或当有错误时返回2
    } finally {
        return 3; // 不合规: 3在1或2之前返回，这是我们没有预料到的
    }
}

合规的解决方案

function foo() {
    try {
        return 1; // 期望返回1
    } catch(err) {
        return 2; //  或当有错误时返回2
    }
}

请参阅

  • MITRE, CWE-584 - Return Inside Finally Block
  • CERT, ERR04-J. - Do not complete abruptly from a finally block
