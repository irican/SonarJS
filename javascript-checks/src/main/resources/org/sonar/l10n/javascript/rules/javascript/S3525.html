最初JavaScript不支持classes，类行为必须使用类似于prototype赋值给"class"函数这样的东西进行组合。幸运的是，ECMAScript 2015增加了类，所以任何残留的prototype使用的都应该转换为真正的classes。新的语法更有表现力和更清晰，尤其是对那些有其他语言经验的人来说。

具体来说，对于ES2015，您应该简单地声明一个类，并在类声明中定义它的方法。

不合规的代码示例

function MyNonClass(initializerArgs = []) {
  this._values = [...initializerArgs];
}

MyNonClass.prototype.doSomething = function () {  // 不合规
  // ...
}

合规的解决方案

class MyClass {
  constructor(initializerArgs = []) {
    this._values = [...initializerArgs];
  }

  doSomething() {
    //...
  }
}
