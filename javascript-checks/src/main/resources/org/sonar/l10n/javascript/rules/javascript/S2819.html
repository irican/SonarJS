HTML5增加了向其他域提供的文档发送消息的能力。根据规格:

在包含任何机密信息的消息中，作者不应该在targetOrigin参数中使用通配符关键字(*)，否则无法保证消息只传递给预期的收件人。

为了降低将敏感信息从敌对域或未知域发送到所服务的文档的风险，此规则在每个时间窗口都会引发一个问题。 Window.postMessage 被使用

不合规的代码示例

var myWindow = document.getElementById('myIFrame').contentWindow;
myWindow.postMessage(message, "*"); // 不合规; 你怎么知道你在myIFrame中加载的东西还在那里?
